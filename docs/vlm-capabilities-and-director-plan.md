# VLM 能力 → 导演计划

> 2026-06-22。一个重要计划:充分利用 VLM 给出的信息,让导演(③ director 层)更好、更灵活地完成任务。
> 现在感知层只把「每一刻的画面描述(caption)+ Whisper 对话 + 每片段人脸名」喂给导演。VLM 还能给更多。

## Qwen2.5-VL(本地模型)在「画面→文字」之外能做什么

| 能力 | 能不能 | 对 composerV / 导演的用处 |
|---|---|---|
| 定位人/物的位置(bounding box / grounding) | ✅ 原生支持,给框坐标 | 自动重构图(横屏→裁到主体)、空间显著入点、「这条多给某人」= 裁到他 |
| 指向(point)/ 计数 | ✅ | 主体跟随、数人头 |
| OCR(读画面里的字) | ✅ | 补上缺失的地点(文件元数据没 GPS,但路牌/招牌能读) |
| 属性描述(穿着 / 表情 / 动作 / 大致年龄) | ✅ | 更细的片段元数据 |
| 构图 / 质量判断(清晰?构图好?) | ✅(主观但可用) | 同一段挑「最好看的一刻」做入点 / 取舍 |
| 人脸 _检测_ + 定位 | ✅ | 配合身份 |
| 人脸 _身份_(「这是谁」) | ❌ 通用 VLM 没有你的人物库 | 身份是 insightface 的活(项目已有家庭库) |

**分工铁律**:VLM =「这一刻有什么 / 在画面哪里 / 什么样」;insightface =「这具体是谁」;Whisper =「说了什么」。三者各司其职,都带时间戳,汇成一张 footage 表喂给导演。

## 已验证(2026-06-22)

grounding 在一帧真实家庭素材上实测:框出 5 个人 + 2 个物体,位置准确。坐标在模型 smart-resize 后的空间(28 的倍数,约 1–2% 偏差),做「裁到主体」够用,要像素级再校正一步。7B 实测约 10 秒/帧(常驻),32B 约 44 秒。脚本:`/tmp/ground_test.py`(临时,逻辑简单可重建)。

## 计划:把这些信号接进导演

- **footage 表扩列**:每个 visual moment 除了 text,再带「人/物的 bbox」「画面上的字(OCR)」「构图质量分」。导演读表时就同时有空间、文字、质量信息。
- **导演据此能做的新动作**(prompt 与 segment 契约要相应扩展):
  - 自动重构图 / 推近主体(横屏素材尤其需要);
  - 有空间感的入点(从主体 / 动作所在处开镜,而非死板从片头或仅靠 motion);
  - 「把某人的镜头都挑出来并裁到他」(VLM 定位 + insightface 身份);
  - 用 OCR 补地点、用质量分挑同一段里最好的一刻。
- **成本**:grounding / OCR 是额外的 VLM 调用(或一次多任务 prompt),增算力 → 用限流办法(`taskpolicy -b` + 片段间冷却,平均占用约 60%)。后续要 benchmark「不吵又尽量快」的平衡点。
- **不动的分工**:身份永远走 insightface;语音永远走 Whisper。

## 导演技能清单(路线图)

把"技能"理解成两类:导演能**做**的剪辑动作(输出侧),和它能**按需调用**的感知工具(输入侧)。状态:✅ 有 · 🟡 设计过/部分 · ⬜ 待建。

### A. 编辑技能(输出侧)

| 技能 | 依赖 | 状态 | 解锁什么 |
|---|---|---|---|
| 取舍 / 排序 / 立意 | footage 表 | ✅ 已是核心 | |
| 长镜 vs 快剪(配速) | feeling | ✅ | |
| 压低音乐 / 顶人声 | 语音表 | ✅ | |
| 整句保留(不切人) | 语音表 | ✅ | |
| 裁到主体 / 重构图(横→竖、推近) | grounding(bbox) | ⬜(能力已验证) | 横屏家庭素材竖版分享、推近孩子 |
| 标题卡 / 角标(日期 / 人名 / 地点) | 日期✅、人脸、OCR | 🟡(日期已有) | 「回顾」:谁、在哪 |
| 交叉溶解 / 转场 | — | 🟡(设计过 L2/L3) | 解决「转场生硬」 |
| match-cut 排序(相似镜头相邻) | shot_type / objects / bbox | 🟡(设计过 L1) | 更顺的流 |
| J/L cut(声音越过画面切点) | 语音表 | 🟡(设计过) | 对话更自然 |
| 变速 / 慢动作、定格 | — | ⬜ | 高光强调 |
| Ken Burns(静帧平移缩放) | bbox(可选) | ⬜ | 给偏静镜头加动感 |
| 段内空镜插入(cutaway) | 表 | ⬜ | 长对话里穿插画面 |

### B. 感知技能(输入侧,按需调用)

把导演从"一次性读表"升级成"会查证的 agent"。

| 工具 | 给导演什么 | 来源 |
|---|---|---|
| 定位主体 / 物(grounding) | bbox | VLM(✅能力) |
| 读屏上文字(OCR) | 文本 → 地点 | VLM |
| 这是谁(身份) | 人名 | insightface ✅ |
| 这段精确说了什么 | 句子 | Whisper ✅ |
| 这一刻好不好看(质量) | 分数 | VLM |
| 放大某区域再看 | 细节 | VLM(裁剪后再跑) |

### 两种安装方式(决定工程量)

- **一次性增强(先做)**:把 B 的结果**预先填进 footage 表** + 给 segment 契约**加字段**(reframe / transition / speed / title)。导演仍是一次出决策,简单可控。
- **智能体(后做)**:导演推理中**按需调 B**(「想推近,先定位孩子」)→ tool-use 循环。更灵活,但要把导演从单次调用改成 agent 循环。

### 建议第一批(高价值、低成本、依赖大多就绪)

1. **裁到主体 / 重构图**(grounding 已验证)—— 横屏家庭素材价值最大;
2. **标题卡**(日期现成;人名走 insightface、地点走 OCR)—— 直接服务「回顾」;
3. **交叉溶解 + match-cut 排序** —— 解决「转场生硬」。

第二批:J/L cut、变速、定格、cutaway。
