story-first · local-first · macOS / Apple Silicon

composerV

Turn a personal video archive into a beat-synced memory reel: local perception (VLM for the picture + Whisper for the audio + Apple Vision for aesthetics), Claude as the editor, zero-render live preview, then hand off to Final Cut Pro for finishing. Footage never leaves your machine.

Run it: uv sync → uv run composerv demo ./composerv-demo (generates fully synthetic demo footage, no downloads) → follow the printed catalog / music index / analyze / montage four-step to make a reel. The director step needs the claude CLI or an ANTHROPIC_API_KEY. Code is MIT; model licenses (including insightface's non-commercial restriction) are in THIRD_PARTY_NOTICES.md.
Four stages strung into one line: ① Ingest catalog → ② Analyze → ③ Director montage → ④ Render preview / export, all revolving around one central SQLite store. The slow perception analysis (VLM for the picture + Whisper for the audio + on-device aesthetic scoring) runs only once and is written into the store; the director only reads the cache each time, so making and re-cutting reels is fast. The director is a single LLM call: it renders the footage into a text footage table, and Claude reads it and returns the edit decisions.

② · submodule AESTHETICS scoring

Implementedinside analyze
Internals
score → tag thresholds (distill_quality): [清晰·构图好] (sharp, well-composed) when score ≥ 0.4 and not a utility frame; [弱/过渡] (weak / transitional) when utility or score ≤ -0.2; the middle band → empty tag.

Planned · music-driven editing (direction 3)

Planned
Idea
A two-pass design: the first pass cuts by emotion and picks a track as usual; after analyzing the chosen track's structure (climax point climax_t + phrase boundaries), a lightweight second-pass director locally reorders while keeping the coarse chronological order and marks the strongest shots; finally a deterministic alignment: slide the music (MusicBed.start_offset_s, without distorting the picture) so the music's climax lands on the strongest shot, and snap the cut points to the phrases.
Status
The design spec has passed adversarial review (docs/superpowers/specs/2026-06-26-music-direction3-design.md); 10 TDD tasks are planned but not yet started.

the director's brain

The director's core prompt

The director does not watch the video; it reads the footage as a text timeline and cuts. The full prompt comes from director/prompt.py, reproduced verbatim.

① PREAMBLE · role and task (prompt.py:16-23)
You are the editor of a personal memory reel — a short, music-backed montage cut from one family's home footage, the kind of video someone keeps to relive a day. You do NOT watch the video; you read it as a timeline of descriptions. Decide the edit: which moments to keep, in what order, how long each runs, and where the music should step back for someone's voice. A human will review and may adjust your cut, so give a short reason for each choice.

INPUT
② HEADER · assembled dynamically per run (prompt.py:85-96)　highlight = runtime variable
- Feeling: {feeling}  (pace: upbeat = quick cuts, calm/nostalgic = longer holds, sad = lingering)
- Music: chosen AFTER your edit to match the energy arc you describe. Cut to the shot lengths the feeling implies; cuts are aligned to the music's beats afterward.
- Target length: about {budget_s}s — be selective; leave out weak material to stay near it.
- Sensitive people (do NOT feature): {sensitive}
- Footage table, chronological. Timestamps are in each clip's OWN seconds. Each clip's rows over time are: 'visual' (what is seen), 'speech' (what is said), and 'on screen' (text read off signs/captions — a clue to place or event, useful for the arc):

{footage_table}
③ RULES + PROCESS + output schema (prompt.py:25-72)　green background = recent changes tied to quality tags / the best moment
HOW TO EDIT (in priority order)
1. Human-led. If a clip carries a human note marking importance or a highlighted span, honour
   it above your own judgement — keep it, and keep what they pointed at.
2. Curate by emotion, not by counting. Keep what is worth remembering — warmth, humour, a
   milestone, a striking sight, something said that matters. Drop filler: logistics, "are you
   filming me", ambient noise, repetitive nothing. Judge this by emotion and memory value, NOT by
   image quality — a treasured moment that is a little soft still belongs; a sharp shot of nothing
   does not. (Quality tags inform the in-point only; see rule 8.)
3. Two dimensions count equally. A moment earns its place by what is SAID (a real exchange) OR
   what is SEEN (a child's first try, a view, a shared activity). Do not keep only talking clips.
4. Keep a worthy conversation WHOLE. If an exchange is worth remembering, keep it complete —
   never cut a person off mid-sentence, and never reduce a conversation to a single line. There
   is no "best line"; keep the moment.
5. Music is the bed. The reel is set to music; keep it in front by default. Only ask to duck it
   (duck_music=true) during a conversation or sound you are deliberately featuring. Do not duck
   the whole reel.
6. Pace by feeling. A visual moment is a short shot at about the feeling's shot length and
   should fall on a beat. A kept conversation is a long hold — let it play; off the beat is fine.
7. Keep chronological order so the day can be relived in sequence. Within that, prefer a smooth
   visual flow (don't jam two jarringly different shots back to back) when the order has slack.
8. Open a visual shot on its active / meaningful instant, not always the clip's start. Some clips
   show a quality hint: a header "best ~Xs" (the sharpest, best-composed instant) and per-row tags
   "[清晰·构图好]" (a strong frame) or "[弱/过渡]" (soft / transitional / filler). Prefer opening a
   shot at or near its best moment, and prefer a "[清晰·构图好]" frame over a "[弱/过渡]" one.
9. Vary the rhythm — mix long holds with quick beats; don't make every shot the same.
10. Fit the length budget by LEAVING THINGS OUT, never by trimming a worthy moment short.
11. Photos are stills (marked [photo …], no sound). Use a good one like a held beat: set in_s=0
    and out_s to a 2-5s hold, kind="photo", and choose motion — "in" (slow push), "out" (slow
    pull) to give it life, or "static". Place it in chronological order with the videos.

PROCESS — two steps, in this order:
A. Think in prose first: what is this footage about, which moments matter and why, the
   emotional arc, what to drop.
B. THEN output the structured edit as a single JSON object, after your reasoning:
{
  "feeling": "<the mood you are cutting to>",
  "arc": "<one or two sentences: the through-line of this reel>",
  "segments": [
    {"clip_id": "<id from the table>", "in_s": 0.0, "out_s": 0.0,
     "kind": "moment | conversation | photo", "motion": "static | in | out (photos only)",
     "duck_music": false, "reason": "<one line>"}
  ]
}
clip_id must be EXACTLY the id shown in [clip <id>] (copy it verbatim, do not shorten it);
in_s/out_s are seconds within that clip, between 0 and its stated len; segments are in final
reel order.

What the director reads is not a video but a text table like this (rendered by director/table.py). One block per clip: a header (id / who is on screen / length / note / best ~Xs), followed by visual / speech / text (on screen) rows interleaved by time. At the same instant the order is fixed: speech → visual → text.

[clip DJI_0022]  who: boy, short hair   len: 20s   note: roasting taro, important   best ~3.2s
  speech  t=1.3-2.1s  is this a taro?
  visual  t=0.0s  roasting food around a charcoal stove  [弱/过渡]
  text    t=0.0s  on screen: Wuyi Mountain Scenic Area
  visual  t=3.2s  close-up of the child picking up the roasted taro  [清晰·构图好]
  speech  t=3.5-5.0s  wow, smells great

[photo IMG_3310]  who: —   (still photo — pick a 2-5s hold via out_s, and motion: static/in/out)
  visual  t=0.0s  the family posing together by the stream

What each field means (click a term for the explanation on the right):

best ~Xs
best moment: the instant with the highest aesthetic score in this clip, a hint to open the in-point here, not an instruction to drop the clip on this basis.
visual
The picture as described by the VLM; the row may end with a [清晰·构图好] or [弱/过渡] quality tag.
speech
Dialogue transcribed by Whisper, with start/end times.
text
On-screen text in the picture (road signs / captions), rendered as on screen: ….
photo
A photo is a still with no sound, a hint to set in_s=0, use out_s for a 2-5s hold, and choose a motion.
Input: footage table + metadata

prompt = PREAMBLE + HEADER + RULES. The dynamic part is assembled by build_director_montage: it reads get_clip_moments_rich (with OCR), clip_aesthetics (to compute the quality tags), and get_transcript from the store, then adds feeling, the music track / tempo, the budget budget_s, and sensitive people.

Then this processing chain (all pure, in director/plan.py + montage.py; click the blue terms for explanations):

director × music

How music gets chosen: the director gives intent, the scorer picks the track

Track selection in the shipping pipeline uses no LLM. The director only says what feeling it wants (a MusicIntent); which specific track is decided by the deterministic scorer rank_tracks. This split is deliberate: the LLM makes the judgement, numbers do the matching, matching the "separate decisions from raw data" principle throughout.

Selection flow · director gives intent → scored selection → beat snap → accounting (click the dashed terms / boxes for explanations)
① Director intent
After reading the footage table, the director outputs a MusicIntent alongside the edit: a 16-point energy_curve + tempo range + mode_pref + valence + target duration. It states what feeling it wants and never names any file. (That line in the director prompt now reads Music: chosen AFTER your edit….)
② Scoring
Every track in the library has offline-computed TrackFeatures. rank_tracks scores each dimension (shape/tempo/valence/mode/duration), weights them by DEFAULT_WEIGHTS, then multiplies by the shape_gate soft threshold. Purely numeric, reproducible, no LLM.
③ Selection
The top-ranked total score becomes the chosen track; if the top is < 0.6 → library_gap, it picks the "least bad" as a fallback and flags that a track should be added.
④ Beat snap
beat_snap_segments uses the chosen track's real beat times to snap each cut point to the nearest beat (adjusting out_sec). Re-running with a different song simply swaps the chosen track and re-snaps to its own beats.
⑤ Accounting
A *.music-rationale.json is written next to the reel: the chosen track + a five-axis breakdown + beat snapping + library_gap, so "why this track" can be reviewed.
Files
director/prompt.py (director outputs MusicIntent) · music/score.py (rank_tracks + weights) · music/features.py (compute features) · music/beatsnap.py (beat snap) · music/library.py (read the feature library)　CLI: composerv music index
In one line: the director gives "what it wants", the scorer gives "which track to use". There is also a separate design-time three-way negotiation (music / director / evaluation) used only to tune the prompt and weights, which never enters the shipping pipeline.

data contracts

Data contracts running through the whole line

The objects actually passed between modules. All timecodes are source-media seconds (float); only FCPXML export converts them to integer frames.

Timeline contract: how an edit decision is representedcomposerv/models.py

IntentionList is the single contract shared by the preview engine and the FCPXML exporter.

class Segment(BaseModel):
    """One item of an IntentionList. clip = a reference to source footage; gap = an intentional hole; photo = a still hold."""
    kind: str = "clip"              # clip | gap | photo
    source_id: str | None = None    # source-clip id; None for a gap
    in_sec: float | None = None
    out_sec: float | None = None
    duration_s: float
    motion: str = ""                # photo: static | in | out (render does a Ken Burns move)
    label: str = ""                 # beat function / marker name
    note: str = ""                  # reason / source, written out as a marker note
    transition_in: str = "hard_cut"
    enabled: bool = True            # off = stays in the document but is left out of the compiled output

class IntentionList(BaseModel):
    story_id: str
    timeline_fps: int = 30
    segments: list[Segment] = []
    music: MusicBed | None = None

class MusicBed(BaseModel):
    """The music bed under the whole timeline plus its ducking rules."""
    path: str
    gain_db: float = 0.0            # music level outside the windows
    duck_db: float = -15.0          # clip source-audio level outside the windows
    fade_out_s: float = 1.5
    highlights: list[AudioHighlight] = []
    music_duck_db: float = -12.0    # how much music is ducked inside a window
    highlight_db: float = 0.0       # source audio raised inside a window

class AudioHighlight(BaseModel):
    """A window where live sound is brought forward and the music steps back. Times are timeline seconds."""
    start_s: float
    end_s: float
    ramp_s: float = 0.40            # fade in / out at both ends
    music_duck_db: float | None = None
    clip_db: float | None = None
    label: str = ""
Perception contract: what the analyze stage cachesanalyze/clip_video.py · store/db.py 
class ClipMoment(BaseModel):           # analyze/clip_video.py
    t: float          # the real sampled timestamp (seconds within the clip)
    text: str         # what happened at that moment
    ocr: str = ""     # on-screen readable text at that moment (signs / captions / place names)
    objects: list[GroundedObject] = []   # object boxes (used by reframe, ignored by the director)

class GroundedObject(BaseModel):
    label: str                 # "person" / "dog" / "sign" ...
    box: list[float]           # [x1,y1,x2,y2], normalized to [0,1]

class ClipAesthetics(BaseModel):         # store/db.py
    best_t: float | None = None
    curve: list[tuple[float, float, bool]] = []   # (t seconds, total score, is-utility)
Director output: the final object handed to rendermusic/montage.py 
class MontagePlan(BaseModel):
    feeling: str = ""
    track: str | None = None        # suggested music file
    tempo: float = 0.0
    label: str = ""                 # which day / part this is, used for naming
    intention: IntentionList        # the edit-decision list actually fed to the render layer
