the director's brain

导演模块核心 Prompt

导演不看视频，它把素材读成一张文字时间线再剪。Prompt 全文来自 director/prompt.py，逐字一致。

① PREAMBLE — 角色与任务（prompt.py:16-23）
You are the editor of a personal memory reel — a short, music-backed montage cut from one family's home footage, the kind of video someone keeps to relive a day. You do NOT watch the video; you read it as a timeline of descriptions. Decide the edit: which moments to keep, in what order, how long each runs, and where the music should step back for someone's voice. A human will review and may adjust your cut, so give a short reason for each choice.

INPUT
② HEADER — 每次运行动态拼接（prompt.py:85-96）　高亮 = 运行时变量
- Feeling: {feeling}  (pace: upbeat = quick cuts, calm/nostalgic = longer holds, sad = lingering)
- Music: chosen AFTER your edit to match the energy arc you describe. Cut to the shot lengths the feeling implies; cuts are aligned to the music's beats afterward.
- Target length: about {budget_s}s — be selective; leave out weak material to stay near it.
- Sensitive people (do NOT feature): {sensitive}
- Footage table, chronological. Timestamps are in each clip's OWN seconds. Each clip's rows over time are: 'visual' (what is seen), 'speech' (what is said), and 'on screen' (text read off signs/captions — a clue to place or event, useful for the arc):

{footage_table}
③ RULES + PROCESS + 输出 schema（prompt.py:25-72）　绿底 = 与美学标签 / best 瞬间相关的最近改动
HOW TO EDIT (in priority order)
1. Human-led. If a clip carries a human note marking importance or a highlighted span, honour
   it above your own judgement — keep it, and keep what they pointed at.
2. Curate by emotion, not by counting. Keep what is worth remembering — warmth, humour, a
   milestone, a striking sight, something said that matters. Drop filler: logistics, "are you
   filming me", ambient noise, repetitive nothing. Judge this by emotion and memory value, NOT by
   image quality — a treasured moment that is a little soft still belongs; a sharp shot of nothing
   does not. (Quality tags inform the in-point only; see rule 8.)
3. Two dimensions count equally. A moment earns its place by what is SAID (a real exchange) OR
   what is SEEN (a child's first try, a view, a shared activity). Do not keep only talking clips.
4. Keep a worthy conversation WHOLE. If an exchange is worth remembering, keep it complete —
   never cut a person off mid-sentence, and never reduce a conversation to a single line. There
   is no "best line"; keep the moment.
5. Music is the bed. The reel is set to music; keep it in front by default. Only ask to duck it
   (duck_music=true) during a conversation or sound you are deliberately featuring. Do not duck
   the whole reel.
6. Pace by feeling. A visual moment is a short shot at about the feeling's shot length and
   should fall on a beat. A kept conversation is a long hold — let it play; off the beat is fine.
7. Keep chronological order so the day can be relived in sequence. Within that, prefer a smooth
   visual flow (don't jam two jarringly different shots back to back) when the order has slack.
8. Open a visual shot on its active / meaningful instant, not always the clip's start. Some clips
   show a quality hint: a header "best ~Xs" (the sharpest, best-composed instant) and per-row tags
   "[清晰·构图好]" (a strong frame) or "[弱/过渡]" (soft / transitional / filler). Prefer opening a
   shot at or near its best moment, and prefer a "[清晰·构图好]" frame over a "[弱/过渡]" one.
9. Vary the rhythm — mix long holds with quick beats; don't make every shot the same.
10. Fit the length budget by LEAVING THINGS OUT, never by trimming a worthy moment short.
11. Photos are stills (marked [photo …], no sound). Use a good one like a held beat: set in_s=0
    and out_s to a 2-5s hold, kind="photo", and choose motion — "in" (slow push), "out" (slow
    pull) to give it life, or "static". Place it in chronological order with the videos.

PROCESS — two steps, in this order:
A. Think in prose first: what is this footage about, which moments matter and why, the
   emotional arc, what to drop.
B. THEN output the structured edit as a single JSON object, after your reasoning:
{
  "feeling": "<the mood you are cutting to>",
  "arc": "<one or two sentences: the through-line of this reel>",
  "segments": [
    {"clip_id": "<id from the table>", "in_s": 0.0, "out_s": 0.0,
     "kind": "moment | conversation | photo", "motion": "static | in | out (photos only)",
     "duck_music": false, "reason": "<one line>"}
  ]
}
clip_id must be EXACTLY the id shown in [clip <id>] (copy it verbatim, do not shorten it);
in_s/out_s are seconds within that clip, between 0 and its stated len; segments are in final
reel order.

导演读到的不是视频，而是这样一张文字表（由 director/table.py 渲染）。每个片段一块：表头（id / 出镜人 / 时长 / 备注 / best ~Xs），其下 visual / speech / text(on screen) 三类行按时间交错。同一时刻顺序固定：speech → visual → text。

[clip DJI_0022]  who: 男孩-短发   len: 20s   note: 烤芋头, 重要   best ~3.2s
  speech  t=1.3-2.1s  这是芋头吗
  visual  t=0.0s  围着炭炉烤食物  [弱/过渡]
  text    t=0.0s  on screen: 武夷山风景区
  visual  t=3.2s  孩子拿起烤好的芋头特写  [清晰·构图好]
  speech  t=3.5-5.0s  哇 好香

[photo IMG_3310]  who: —   (still photo — pick a 2-5s hold via out_s, and motion: static/in/out)
  visual  t=0.0s  一家人在溪边合影

各字段含义（点术语看右侧解释）：

最佳瞬间：该片美学得分最高的时刻，提示导演把入点开在这里，不是叫它据此删片。
visual
VLM 描述的画面；行尾带 [清晰·构图好] 或 [弱/过渡] 质量标签。
speech
Whisper 转写的台词，带起止时间。
text
画面里的 屏上文字（路牌/字幕），渲染成 on screen: …。
photo
照片是静帧无声，提示设 in_s=0、用 out_s 定 2-5s 停留并选 运动。
输入：footage table + 元数据

director × music

配乐怎么被选中：导演给意图，打分器选曲

出货管线里选曲没有 LLM。导演只说「要什么感觉」（一个 MusicIntent），具体哪一首由确定性打分器 rank_tracks 定。这是刻意的切分：LLM 负责判断，数值负责匹配，呼应全线「决策与原始数据分离」。

选曲流程 · 导演出意图 → 打分选曲 → 卡拍 → 记账（虚线/方块可点开看解释）
① 导演意图
导演读素材表后，除了剪辑还输出一个 MusicIntent：16 点 energy_curve + tempo 区间 + mode_pref + valence + 目标时长。它说清楚「要什么感觉」，不点名任何一首文件。（导演 prompt 里那行现在写的是 Music: chosen AFTER your edit…）
② 打分
库里每首曲子有离线算好的 TrackFeatures。rank_tracks 逐项打分（shape/tempo/valence/mode/duration），按 DEFAULT_WEIGHTS 加权，再乘 shape_gate 软门槛。纯数值、可复现、无 LLM。
③ 选中
取总分第一名为选中曲子；若第一名 < 0.6 → library_gap，挑「最不差」的兜底并标记该补曲。
④ 卡拍
beat_snap_segments 用选中曲子的真实拍点把每个切点吸到最近拍（改 out_sec）。换一首歌重跑，就是换选中曲子后按它自己的拍点重卡。
⑤ 记账
成片旁写 *.music-rationale.json：选中曲子 + 五轴 breakdown + 拍点吸附 + library_gap，方便复核「为什么配这首」。
文件
director/prompt.py（导演出 MusicIntent） · music/score.py（rank_tracks + 权重） · music/features.py（算特征） · music/beatsnap.py（卡拍） · music/library.py（读特征库）　CLI: composerv music index
一句话：导演给「想要什么」，打分器给「用哪首」。另有一套设计时三方协商（配乐/导演/评估）只用来调提示词和权重，绝不进出货管线。

贯穿全线的数据契约

模块之间真正传递的对象。所有时间码都是源媒体秒（float），只有导出 FCPXML 时才换算成整数帧。

时间线契约：剪辑决策怎么表示composerv/models.py

IntentionList 是预览引擎和 FCPXML 导出器共用的唯一契约。

class Segment(BaseModel):
    """IntentionList 的一项。clip=引用源片；gap=故意的空洞；photo=静帧停留。"""
    kind: str = "clip"              # clip | gap | photo
    source_id: str | None = None    # 源片 id；gap 为 None
    in_sec: float | None = None
    out_sec: float | None = None
    duration_s: float
    motion: str = ""                # 照片: static | in | out（渲染做 Ken Burns）
    label: str = ""                 # beat 功能 / 标记名
    note: str = ""                  # 理由 / 出处，落成 marker 备注
    transition_in: str = "hard_cut"
    enabled: bool = True            # 关掉=留在文档但不进编译输出

class IntentionList(BaseModel):
    story_id: str
    timeline_fps: int = 30
    segments: list[Segment] = []
    music: MusicBed | None = None

class MusicBed(BaseModel):
    """铺在整条时间线下的配乐 + ducking 规则。"""
    path: str
    gain_db: float = 0.0            # 窗口外音乐音量
    duck_db: float = -15.0          # 窗口外片段原声音量
    fade_out_s: float = 1.5
    highlights: list[AudioHighlight] = []
    music_duck_db: float = -12.0    # 窗口内压低音乐的量
    highlight_db: float = 0.0       # 窗口内抬起的原声

class AudioHighlight(BaseModel):
    """现场声被抬到前面、音乐让位的窗口。时间是时间线秒。"""
    start_s: float
    end_s: float
    ramp_s: float = 0.40            # 两端淡入淡出
    music_duck_db: float | None = None
    clip_db: float | None = None
    label: str = ""
感知契约：分析阶段缓存了什么analyze/clip_video.py · store/db.py 
class ClipMoment(BaseModel):           # analyze/clip_video.py
    t: float          # 真实采样到的时间戳（片内秒）
    text: str         # 那一刻发生了什么
    ocr: str = ""     # 那一刻屏上可读文字（路牌/字幕/地名）
    objects: list[GroundedObject] = []   # 物体框（给 reframe 用，导演忽略）

class GroundedObject(BaseModel):
    label: str                 # "person" / "dog" / "sign" ...
    box: list[float]           # [x1,y1,x2,y2]，归一化到 [0,1]

class ClipAesthetics(BaseModel):         # store/db.py
    best_t: float | None = None
    curve: list[tuple[float, float, bool]] = []   # (t秒, 总分, 是否utility)
